## 快速开始

### 5 分钟快速体验

```bash
# 1. 克隆项目
git clone https://github.com/NotSleeply/Nova-Compiler.git
cd Nova-Compiler

# 2. 构建编译器
make

# 3. 测试编译器
./bin/novac --help

# 4. 编译一个示例
./bin/novac --codegen examples/minimal.nv

# 5. 编译生成的 C 代码
gcc -o minimal output.c
./minimal
```

## 安装和构建

### 系统要求

- **操作系统**: Windows 10+, Linux, macOS 10.15+
- **编译器**: GCC 9+ 或 Clang 10+ (支持 C++17)
- **构建工具**: GNU Make 4.0+
- **可选**: Git (用于版本控制)

### 安装步骤

#### Windows (MinGW)

```bash
# 1. 安装 MinGW-w64 (包含 g++ 和 make)
# 下载地址: https://www.mingw-w64.org/

# 2. 验证安装
g++ --version    # 需要 9.0 或更高
make --version

# 3. 克隆并构建
git clone https://github.com/NotSleeply/Nova-Compiler.git
cd Nova-Compiler
make
```

#### Linux (Ubuntu/Debian)

```bash
# 1. 安装依赖
sudo apt update
sudo apt install build-essential g++-9

# 2. 克隆并构建
git clone https://github.com/NotSleeply/Nova-Compiler.git
cd Nova-Compiler
make
```

#### macOS

```bash
# 1. 安装 Xcode Command Line Tools
xcode-select --install

# 2. 安装 Homebrew (如果需要)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 3. 安装 GCC
brew install gcc

# 4. 克隆并构建
git clone https://github.com/NotSleeply/Nova-Compiler.git
cd Nova-Compiler
make
```

### 验证安装

## 基本使用

### 命令行选项

```bash
novac [选项] <源文件>
```

| 选项 | 长选项 | 说明 |
|------|--------|------|
| `-h` | `--help` | 显示帮助信息 |
| `-v` | `--version` | 显示版本信息 |
| `-l` | `--lex` | 仅运行词法分析，打印词元 |
| `-p` | `--parse` | 仅运行语法分析，打印 AST |
| `-s` | `--semantic` | 运行语义分析 |
| | `--ir` | 生成并打印中间表示 |
| | `--codegen` | 生成目标代码 |
| `-o` | `--output FILE` | 指定输出文件名 |

### 典型工作流程

#### 1. 词法分析 (Lexing)

将源代码分解为词元（tokens）：

```bash
./bin/novac --lex examples/minimal.nv
```

**输出示例:**
```
=== Tokens ===

KW_FN 'fn' at examples/minimal.nv:1:1
IDENTIFIER 'add' at examples/minimal.nv:1:4
LPAREN '(' at examples/minimal.nv:1:7
IDENTIFIER 'a' at examples/minimal.nv:1:8
COLON ':' at examples/minimal.nv:1:9
KW_INT 'int' at examples/minimal.nv:1:11
COMMA ',' at examples/minimal.nv:1:14
IDENTIFIER 'b' at examples/minimal.nv:1:16
COLON ':' at examples/minimal.nv:1:17
KW_INT 'int' at examples/minimal.nv:1:19
RPAREN ')' at examples/minimal.nv:1:22
ARROW '->' at examples/minimal.nv:1:24
KW_INT 'int' at examples/minimal.nv:1:27
LBRACE '{' at examples/minimal.nv:1:31
KW_RETURN 'return' at examples/minimal.nv:2:5
IDENTIFIER 'a' at examples/minimal.nv:2:12
OP_PLUS '+' at examples/minimal.nv:2:14
IDENTIFIER 'b' at examples/minimal.nv:2:16
SEMICOLON ';' at examples/minimal.nv:2:17
RBRACE '}' at examples/minimal.nv:3:1
EOF '' at examples/minimal.nv:3:2
```

#### 2. 语法分析 (Parsing)

将词元流构建为抽象语法树：

```bash
./bin/novac --parse examples/minimal.nv
```

**输出示例:**
```
=== Abstract Syntax Tree ===

Program: examples/minimal.nv
  FunctionDecl: add
    Parameters:
      ParamDecl: a : PrimitiveType: int
      ParamDecl: b : PrimitiveType: int
    Return Type: PrimitiveType: int
    Body:
      BlockStmt:
        ReturnStmt:
          BinaryExpr:
            Identifier: a
            Op: OP_PLUS
            Identifier: b
```

#### 3. 语义分析 (Semantic Analysis)

检查类型正确性和语义错误：

```bash
./bin/novac --semantic examples/minimal.nv
```

**输出示例:**
```
=== Semantic Analysis ===

✓ No semantic errors found

Symbol Table:
  Function: add(int, int) -> int
    Parameter: a : int
    Parameter: b : int
```

#### 4. 中间表示生成 (IR Generation)

生成三地址码形式的中间表示：

```bash
./bin/novac --ir examples/minimal.nv
```

**输出示例:**
```
=== Intermediate Representation ===

Module: examples/minimal.nv
  Global Variables: (none)
  
  Function: add(int, int) -> int
    Parameters: a, b
    
    Basic Block: entry
      %t0 = a + b
      return %t0
```

#### 5. 代码生成 (Code Generation)

生成目标 C 代码：

```bash
./bin/novac --codegen -o output.c examples/minimal.nv
```

**生成的 output.c:**
```c
/* Generated by Nova Compiler */
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <stdbool.h>

int add(int a, int b) {
    /* Temporary variables */
    int _t_t0;

    /* Block: entry */
    entry:
        _t_t0 = a + b;
        return _t_t0;
}
```

## 高级功能

### 编译并执行程序

完整的编译和执行流程：

```bash
# 1. 编译 Nova 代码到 C
./bin/novac --codegen -o program.c source.nv

# 2. 使用 GCC 编译 C 代码
gcc -o program program.c

# 3. 执行生成的程序
./program
```

### 一键编译脚本

创建一个脚本 `compile.sh`:

```bash
#!/bin/bash
# Nova 一键编译脚本

SOURCE=$1
OUTPUT=${2:-program}

# 编译 Nova 到 C
./bin/novac --codegen -o ${OUTPUT}.c ${SOURCE}
if [ $? -ne 0 ]; then
    echo "❌ Nova 编译失败"
    exit 1
fi

# 编译 C 到可执行文件
gcc -o ${OUTPUT} ${OUTPUT}.c
if [ $? -ne 0 ]; then
    echo "❌ C 编译失败"
    exit 1
fi

echo "✅ 编译成功: ${OUTPUT}"
./${OUTPUT}
```

**使用方法:**
```bash
chmod +x compile.sh
./compile.sh examples/calculator.nv mycalc
```

### 调试模式

查看每个编译阶段的详细信息：

```bash
# 查看所有阶段的输出
./bin/novac --lex source.nv > tokens.txt
./bin/novac --parse source.nv > ast.txt
./bin/novac --semantic source.nv > semantic.txt
./bin/novac --ir source.nv > ir.txt
./bin/novac --codegen source.nv > code.c
```

## 示例教程

### 示例 1: Hello World

**源代码 (hello.nv):**
```nova
fn greet(name: string) -> string {
    return "Hello, " + name + "!";
}

fn main() -> void {
    let message: string = greet("World");
    println(message);
}
```

**编译和运行:**
```bash
./bin/novac --codegen hello.nv
gcc -o hello hello.nv.c
./hello
```

**预期输出:**
```
Hello, World!
```

### 示例 2: 阶乘计算

**源代码 (factorial.nv):**
```nova
fn factorial(n: int) -> int {
    if (n <= 1) {
        return 1;
    }
    return n * factorial(n - 1);
}

fn main() -> void {
    let result: int = factorial(5);
    println(result);  // 输出 120
}
```

**编译和运行:**
```bash
./bin/novac --codegen factorial.nv
gcc -o factorial factorial.nv.c
./factorial
```

**预期输出:**
```
120
```

### 示例 3: 计算器

**源代码 (calculator.nv):**
```nova
fn add(a: int, b: int) -> int {
    return a + b;
}

fn subtract(a: int, b: int) -> int {
    return a - b;
}

fn multiply(a: int, b: int) -> int {
    return a * b;
}

fn divide(a: int, b: int) -> int {
    if (b == 0) {
        return 0;  // 错误处理
    }
    return a / b;
}

fn main() -> void {
    let x: int = 10;
    let y: int = 3;
    
    println(add(x, y));       // 13
    println(subtract(x, y));  // 7
    println(multiply(x, y));  // 30
    println(divide(x, y));    // 3
}
```

## 故障排除

### 常见编译错误

#### 1. "undefined reference to main"

**原因:** Nova 程序没有 `main` 函数。

**解决方案:**
```nova
// 添加 main 函数
fn main() -> void {
    // 你的代码
}
```

#### 2. "type mismatch"

**原因:** 类型不匹配，例如将 `int` 赋值给 `string`。

**解决方案:**
```nova
// 错误示例
let x: int = "hello";  // ❌

// 正确示例
let x: int = 42;       // ✅
let y: string = "hello";  // ✅
```

#### 3. "undefined variable"

**原因:** 使用了未声明的变量。

**解决方案:**
```nova
// 错误示例
fn test() -> void {
    println(x);  // ❌ x 未定义
}

// 正确示例
fn test() -> void {
    let x: int = 42;
    println(x);  // ✅
}
```

### 构建问题

#### Windows: "g++: command not found"

**解决方案:**
```bash
# 确认 MinGW 已安装并添加到 PATH
where g++

# 如果未找到，添加到环境变量
# PATH: C:\MinGW\bin
```

#### Linux: "fatal error: bits/c++config.h: No such file"

**解决方案:**
```bash
# 安装完整的 C++ 标准库
sudo apt install g++-9-multilib
```

### 运行时问题

#### 生成的程序立即崩溃

**可能原因:**
1. 除零错误
2. 数组越界
3. 无限递归导致栈溢出

**调试方法:**
```bash
# 使用 gdb 调试
gcc -g -o program program.c
gdb ./program
(gdb) run
(gdb) bt  # 查看调用栈
```

## 常见问题

### Q1: Nova 支持哪些平台？

**A:** Nova 编译器本身可在 Windows、Linux 和 macOS 上运行。生成的 C 代码可编译为任何平台，具有高度可移植性。

### Q2: Nova 是解释型还是编译型语言？

**A:** Nova 是编译型语言，编译流程为：
```
Nova 源码 → AST → IR → C 代码 → 机器码
```

### Q3: 如何处理字符串？

**A:** Nova 支持 C 风格的字符串：
```nova
let greeting: string = "Hello, World!";
let name: string = "Nova";
let message: string = greeting + " from " + name;
```

### Q4: 支持哪些数据类型？

**A:** 当前支持：
- `int` - 64 位整数
- `float` - 双精度浮点数
- `bool` - 布尔值 (`true`/`false`)
- `string` - 字符串
- `char` - 字符
- `void` - 无类型

### Q5: 如何调试 Nova 程序？

**A:** 使用 `--lex`、`--parse`、`--semantic` 和 `--ir` 选项查看每个阶段的输出，定位问题。

### Q6: 可以使用外部库吗？

**A:** 目前不支持直接使用外部库，但可以通过 FFI (Foreign Function Interface) 在生成的 C 代码中调用 C 库。

### Q7: 性能如何？

**A:** Nova 编译生成的 C 代码性能接近手写的 C 代码，因为：
- 静态类型检查
- 优化的 IR 生成
- 利用 C 编译器的优化

### Q8: 有 IDE 支持吗？

**A:** 目前没有专门的 IDE 插件，但可以使用任何文本编辑器编写 Nova 代码，如：
- VS Code (配合语法高亮插件)
- Vim/Neovim
- Sublime Text

### Q9: 如何贡献代码？

**A:** 请参考 [CONTRIBUTING.md](CONTRIBUTING.md)，步骤：
1. Fork 项目
2. 创建功能分支
3. 提交 Pull Request

### Q10: 项目路线图是什么？

**A:** 
- v0.2.0 (当前): 完整前端
- v0.3.0: 优化器
- v0.4.0: 多后端支持 (LLVM, x86-64, WASM)
- v1.0.0: 生产就绪版本
